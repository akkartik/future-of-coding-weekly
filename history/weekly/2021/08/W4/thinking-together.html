Konrad Hinsen 2021-08-19 10:43:30

I agree with Ivan Reese that XML is too complex (in spite of being the simplification of SGML, which is perhaps the ideal example for showcasing overengineering) and JSON too simple. An interesting intermediate is STON (Smalltalk Object Notation), which is indeed the idea of JSON applied to a language with a richer data model.

But there is another, perhaps orthogonal difference. XML was never meant to be a dump of some data structure. It always includes some metadata, in the form of tags. Pretty much all XML documents contain information that you could just throw away if you only care about data exchange between two software agents.

William Taysom 2021-08-19 11:51:10

One headache with XML is that a 90% parser is roughly as tricky as JSON, but that last 10% is a lot of work for little benefit.

Another headache is more accidental complexity. Do you go with <tag property="value">more stuff</tag> or <tag><property>value</property><more>more stuff</more></tag> or whatever else? JSON has this quality too, but to a lesser degree.
