Andreas S. 2020-08-25 07:07:51

Hi #present-company  ! Could someone explain or give links to resources that compare MVVM and the react Approach? How do they compare and relate? Thanks!

Emmanuel Oga 2020-08-25 07:25:06

I don't know about a direct comparison but from the top of my head one of the pioneers of mvvm on the web was https://knockoutjs.com/ One possible way to learn about MVVM is to go ahead and build a simple app with it. I suspect you probably already did some react already? The gist of mvvm is to manually build a graph of dependencies, and when you change the data from code, it should directly modify the DOM attached to some end of the graph you are creating. Changes are "bidirectional": when you change the data, it changes the DOM, and when you change the DOM, it changes the (JS) data.

Emmanuel Oga 2020-08-25 07:26:03

my experience with MVVM (and KJS) was that the graphs ended up being a bit hairy and it was easy to accidentally create loops, but it was hard to track exactly where the loop was

Emmanuel Oga 2020-08-25 07:27:00

the graphs were not reified, you couldn't. say, generate a png with graphviz of what your dependencies looked like

Emmanuel Oga 2020-08-25 07:28:44

react does away from that by making the data always "flow" a single direction, from your POV the DOM is kind of a side effect, always destroyed and recreated from the data each time the data changes (and the framework does the diff and patch "trick" to make the process efficient)

Andreas S. 2020-08-25 11:26:44

Ryan King 2020-08-25 15:48:24

The way I understand things is React is just the view, and the view is defined as a function of state. The state, however, can be implemented however you choose.

You could totally implement MVVM using React as the view layer. My personal setup involves some observable models in which React components listen for changes and re-render in such event - which doesn't seem too far removed from MVVM in my head at least.

nicolas decoster 2020-08-25 17:29:38

Vue.js is a good example of current MVVM for the web. The View is the Vue.js template, and the ViewModel is the Vue.js data.

React is not MVVM by default but I guess you can do MVVM with it, as Ryan said.
